**Техническое задание (ТЗ)**  
**Система управления персоналом (HRMS) с доской Kanban**  
**Стек: TypeScript, Vue 3 (Composition API), Pinia, Vue Router, Tailwind CSS**

### 1. Общие сведения

- Название проекта: HRFlow  
- Тип: Веб-приложение (SPA)  
- Цель: Централизованное управление сотрудниками, процессами HR и задачами в формате Kanban  
- Аудитория: HR-специалисты, руководители, сотрудники компании  
- Язык интерфейса: Русский (с возможностью добавления английского)  
- Адаптивность: Полностью responsive (мобильные + планшеты + десктоп)

### 2. Основные модули / разделы

1. **Аутентификация и пользователи**  
   - Регистрация / вход / восстановление пароля  
   - Роли: Admin, HR, Manager, Employee  
   - RBAC (Role-Based Access Control) через Pinia + guards в роутере  

2. **Сотрудники (Employee Directory)**  
   - Список сотрудников (таблица + карточки)  
   - Фильтры: отдел, должность, статус, поиск  
   - Карточка сотрудника:  
     - Личные данные (ФИО, дата рождения, телефон, email, фото)  
     - Трудовые данные (должность, отдел, руководитель, дата приема/увольнения, тип занятости)  
     - Документы (сканы паспорта, трудовой, СНИЛС и т.д. — загрузка файлов)  
     - История изменений  

3. **Штатное расписание и оргструктура**  
   - Дерево подразделений  
   - Штатные единицы (позиции)  
   - Вакансии  

4. **Кандидаты / Рекрутинг**  
   - Воронка подбора (Kanban по умолчанию)  
   - Этапы: Новая заявка → Собрано резюме → Телефонное интервью → Интервью → Оффер → Отказ / Принят  
   - Карточка кандидата: резюме, комментарии, прикрепленные файлы, история перемещений  

5. **Онбординг / Адаптация**  
   - Kanban-доска для каждого нового сотрудника  
   - Типовые чек-листы: IT, бухгалтерия, руководитель, HR, охрана труда  
   - Назначение ответственных, дедлайны, статусы  

6. **Задачи и процессы HR (главная Kanban-доска)**  
   - Единая доска задач HR-отдела  
   - Колонки (настраиваемые):  
     - Backlog / Новые  
     - To Do  
     - В работе  
     - На проверке  
     - Готово / Архив  
   - Типы задач:  
     - Заявка на отпуск  
     - Увольнение  
     - Изменение оклада  
     - Выдача оборудования  
     - Обучение  
     - Корпоративное мероприятие  
     - и т.д.  
   - Поля карточки задачи: заголовок, описание, тип, приоритет, дедлайн, ответственный, связанные сотрудники/кандидаты, вложения, комментарии, чек-лист, история  

7. **Отпуска и отсутствия**  
   - Календарь отпусков (общий + личный)  
   - Заявки на отпуск (согласование через Kanban или линейный workflow)  
   - Баланс дней отпуска  

8. **Оценка и развитие**  
   - Цели и KPI (OKR / цели на период)  
   - Оценка 360 / Performance review  
   - Планы развития  

9. **Отчеты и аналитика**  
   - Дашборд: headcount, текучесть, возрастной/гендерный состав  
   - Отчеты: по отпускам, по найму, по эффективности подбора  
   - Экспорт в XLSX/CSV/PDF  

10. **Настройки системы** (только Admin)  
    - Управление ролями и правами  
    - Настройка колонок Kanban  
    - Справочники (отделы, должности, типы задач, причины увольнения и т.д.)  
    - Шаблоны задач и чек-листов  

### 3. Технические требования

**Frontend**  
- Vue 3 + Composition API  
- TypeScript (строгий режим, noImplicitAny)  
- State management: Pinia (stores по модулям: auth, employees, kanban, etc.)  
- Routing: Vue Router 4 (history mode) + route meta для прав доступа  
- Стили: Tailwind CSS + PostCSS  
- UI-библиотека: Headless UI + Heroicons (или аналог без тяжелых зависимостей)  
- Формы: VueUse + Zod для валидации  
- Таблицы: TanStack Table / Vue Good Table  
- Drag & Drop для Kanban: @dnd-kit или sortablejs  
- Дата/время: date-fns  
- HTTP: Axios + interceptors (auth token, errors)  
- Хранение токена: HttpOnly cookie или localStorage (с защитой)  

**Backend** (указать в ТЗ как отдельный пункт)  
- REST API или GraphQL (рекомендуется REST)  
- Аутентификация: JWT  
- База данных: PostgreSQL / MongoDB  
- Файлы: S3-совместимое хранилище  

### 4. Нефункциональные требования

- Производительность: загрузка списка до 1000 сотрудников < 2 сек  
- Безопасность: HTTPS, защита от XSS/CSRF, rate limiting, хэширование паролей  
- Аудит: логи важных действий (кто создал/переместил задачу, изменил данные сотрудника)  
- Мобильная версия: полный функционал на экранах ≥ 320px  
- Доступность (a11y): ARIA-атрибуты, контрастность ≥ 4.5:1  
- SEO: SSR не требуется, но meta-теги и заголовки обязательны  
- Тестирование: unit (Vitest) ≥ 70% покрытие ключевых stores и компонентов  
- Документация: README + Storybook для компонентов  

### 5. Приемка и этапы

- Этап 1: Аутентификация + профиль + список сотрудников  
- Этап 2: Kanban-доска (задачи + drag&drop)  
- Этап 3: Модули Рекрутинг + Онбординг  
- Этап 4: Отпуска, оргструктура, отчеты  
- Финал: Роли, настройки, polish
